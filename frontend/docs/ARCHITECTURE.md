```markdown
# Архитектура проекта WayAtom

## Общая структура
src/
components/ # Переиспользуемые React-компоненты (карта, чат, сайдбар и др.)
Map/ # Компоненты, связанные с картой (Leaflet)
YandexMap/ # Компоненты для Яндекс.Карт (если используются)
...
pages/ # Страницы приложения (Home, Profile, Planner и др.)
types/ # TypeScript типы и интерфейсы (например, marker.ts)
services/ # Сервисы для работы с API, внешними библиотеками и т.д.
styles/ # Глобальные и модульные стили (если есть)
assets/ # Статические ресурсы (изображения, иконки и т.д.)
hooks/ # Кастомные React-хуки
contexts/ # React Contexts для глобального состояния
utils/ # Утилиты и вспомогательные функции
database/ # SQL-скрипты и миграции (если есть)
config/ # Конфигурационные файлы
i18n/ # Локализация
layouts/ # Общие layout-компоненты
App.tsx # Корневой компонент приложения
main.tsx # Точка входа приложения
index.css # Глобальные стили, Tailwind, импорты CSS библиотек

## Основные технологии

- **React** — UI библиотека
- **TypeScript** — типизация
- **Leaflet** — карта и маркеры
- **leaflet.markercluster** — кластеризация маркеров
- **Tailwind CSS** и **styled-components** — стилизация
- **Vite** — сборщик проекта
- **Font Awesome** — иконки
- **Express/Node.js** — серверная часть (API)
- **SQL** — база данных

## Ключевые компоненты

- **Map** — отображение карты, маркеров, кластеризация, попапы (Leaflet)
- **MarkerPopup** — кастомный попап для маркеров (React-портал)
- **AddMarkerButton** — добавление новых меток
- **MapLegend** — легенда карты
- **Sidebar** — боковое меню для фильтрации и управления
- **Chat** — чат между пользователями
- **Profile** — страница профиля пользователя
- **Planner** — планировщик маршрутов/поездок

## Работа с маркерами

- Маркеры берутся с API (`/api/markers`)
- Для каждого маркера определяется категория, цвет, иконка, "горячесть" (hot)
- Кластеризация реализована через leaflet.markercluster
- Попапы реализованы через React-портал и кастомные компоненты
- В попапе доступны действия: лайк, комментарии, поделиться, маршрут, избранное, настройки и др.

## Стилизация

- Tailwind CSS для большинства компонентов
- styled-components для кастомных стилей (особенно для карты и попапов)
- Глобальные стили и стили для карты — в `index.css` и `Map.styles.ts`
- Импорт стилей Leaflet и MarkerCluster — через JS/TS

## API

- Все данные (например, маркеры) получаются через REST API (`/api/markers` и др.)
- Сервисы для работы с API лежат в папке `services/`
```
